# Описание структуры проекта API диссертационного совета

## Основные компоненты

### `main.py`
- Точка входа приложения FastAPI.
- Подключает CORS middleware, lifespan для управления подключением к БД.
- Регистрирует все CRUD-роутеры из пакета `routers`.
- Содержит три эндпоинта для работы с полной формой диссертации:
  - `POST /api/thesis/form-data` – создание новой диссертации и всех связанных записей из JSON.
  - `POST /api/thesis/{thesis_id}/form-data` – обновление существующей диссертации.
  - `GET /api/thesis/{thesis_id}/form-data` – получение всех данных формы, собранных из БД.

### `models.py`
- Описывает Pydantic-модели для валидации входящих данных:
  - `ThesisFormData` – полный набор полей формы (400+).
  - CRUD-модели: `PersonCreate/Update/Out`, `Organization*`, `Specialty*`, `Council*`, `Thesis*`, `Employment*`, `Education*`, `Achievement*`, `ApplicantDetails*`, `Opponent*`, `Participation*`, `DsEvent*` и др.

### `database.py`
- Конфигурация подключения к PostgreSQL через `databases` и `sqlalchemy`.
- Объект `database` используется всеми роутерами для выполнения асинхронных запросов.

### `utils.py`
- Вспомогательные функции:
  - `parse_full_name()` – разбор строки "Фамилия Имя Отчество" на компоненты.
  - `parse_russian_date()` – парсинг дат в форматах «1 января 2025», «DD.MM.YYYY», ISO.
  - `extract_year()` – извлечение года из строки даты.

### `field_mapping.py`
- *Устаревший файл.* Изначально содержал маппинг полей формы на таблицы БД для прямых SQL-запросов. Больше не используется, заменён на `form_handler.py`.

### `form_handler.py`
- **Основной обработчик создания/обновления диссертации.** Содержит функцию `process_thesis_form(thesis_id, form_data)`, которая последовательно:
  1. Получает контекст диссертации (все ID).
  2. Обновляет основную таблицу `thesis` (название, степень, даты и т.д.).
  3. Обновляет совет, степени председателя и секретаря.
  4. Сохраняет данные соискателя (персона, детали, место работы, образование).
  5. Сохраняет научного руководителя (персона, место работы, статьи).
  6. Сохраняет оппонентов (персоны, места работы, статьи).
  7. Сохраняет достижения соискателя (статьи, конференции, тезисы, РИД) в `achievement` и связи `thesis_achievement`.
  8. Сохраняет контентные блоки (задачи, новизна, паспорт специальности, завкафедрой, декан) в `thesis_content_block`.
  9. Сохраняет события (семинар, создание комиссии, приём к защите) в `ds_event`.
- Использует готовые CRUD-функции из роутеров (`create_person`, `update_person`, `create_employment` и т.д.), избегая прямых SQL-запросов для большинства операций.

### `data_assembler.py`
- Содержит `assemble_full_thesis_json(thesis_id)` – собирает полный словарь со всеми полями формы, извлекая данные из всех связанных таблиц:
  - `thesis` + `dissertation_council` – основная информация.
  - `person` – имена, степени (соискатель, руководитель, оппоненты, председатель, секретарь).
  - `applicant_details` – паспортные данные, адрес, СНИЛС.
  - `employment_history` – места работы.
  - `education_history` – образование (высшее и аспирантура).
  - `specialty` – коды и названия специальностей.
  - `achievement` + `thesis_achievement` – статьи/конференции/тезисы/РИД.
  - `thesis_content_block` – задачи, новизна, ценность, паспорт специальности, завкафедрой, декан.
  - `ds_event` – протоколы (семинар, комиссия, защита).
- Возвращает словарь, совместимый с моделью `ThesisFormData`, где все пропущенные поля — пустые строки.

### `routers/`
- Пакет с отдельными FastAPI-роутерами для CRUD-операций над сущностями:
  - `persons.py`, `organizations.py`, `specialties.py`, `councils.py`, `theses.py`, `achievements.py`, `employments.py`, `educations.py`, `applicant_details.py`, `opponents.py`, `participations.py`, `events.py`.
- Каждый роутер содержит стандартные эндпоинты: `GET /`, `GET /{id}`, `POST /`, `PUT /{id}`, `DELETE /{id}`.
- Используются в `main.py` и также вызываются функциями из `form_handler.py` для повторного использования логики.

### `index.html`
- Одностраничный интерфейс на React (подключается через Babel для JSX).
- **Функциональность**:
  - Пошаговый ввод всех 400+ полей формы, сгруппированных по файлам и секциям.
  - Кнопка «Загрузить из БД» – выполняет `GET /api/thesis/{id}/form-data` и заполняет поля.
  - Кнопка «Отправить запрос» – отправляет `POST /api/thesis/{id}/form-data` (обновление) или `POST /api/thesis/form-data` (создание) с полным JSON.
  - После успешной отправки автоматически генерируются и скачиваются TeX-файлы (шаблоны) на основе заполненных полей.

## Взаимодействие

1. **Пользователь** открывает `index.html`, заполняет поля (или загружает существующую диссертацию по ID).
2. При нажатии «Отправить запрос» фронтенд отправляет JSON на сервер.
3. **FastAPI** валидирует JSON через модель `ThesisFormData`.
4. `main.py` вызывает `process_thesis_form` из `form_handler.py`.
5. Обработчик создаёт/обновляет записи во всех таблицах, используя CRUD-роутеры.
6. При нажатии «Загрузить из БД» фронтенд вызывает `GET /api/thesis/{id}/form-data`.
7. `data_assembler.py` собирает данные из всех связанных таблиц и возвращает полный JSON.
8. Фронтенд обновляет поля формы.

## Тестовый скрипт (`create_test_thesis.py`)
- Отдельный Python-скрипт для массового тестирования.
- Отправляет POST-запрос на `/api/thesis/form-data` со всеми заполненными полями.
- Может быть использован для быстрой проверки сохранения и загрузки всех данных.
